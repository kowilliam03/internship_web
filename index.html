學習

由於研究主題涵蓋多個領域，因此每個牽涉到的領域都需要有基本的認識，以下會簡單羅列出相關的領域，以其學到的知識，詳細內容會放在書面報告中。

Docker

  • Docker是實作輕量級的作業系統虛擬化的解決方案。
  • Docker的基礎是Linux容器(Linux Containers, LXC)等技術。
  • 學習如何將撰寫好的服務包裝成Docker，並部署至雲端(AWS、Azure)

    • Microservices(微服務)

  • 了解微服務的架構，以及微服務對資訊界產生的影響。
  • 使用基於Python開發的輕量級Web應用框架:Flask，搭建微服務的應用。

    • Data Analysis(資料分析)

  • 了解資料分析的流程，包含:EDA、資料視覺化。
  • 主要使用:Pandas及Numpy進行分析資料，matplotlib及seaborn進行資料視覺化

    • Machine Learning(機器學習)

  • 將指導老師的機器學習講義英翻中，翻譯的過程中，也學到不少東西，像是:機器學習的流程、 EDA的重要性等等。

    • Computer Vision(電腦視覺)

  • 了解Image Processing(影像處理)的基本概念，像是:Filter、Threshold、Edge detection、 Object detection等基本原理及相關的演算法。
  • 使用Scikit-Image套件進行影像處理。

    • 雲端服務

  • 了解雲端服務帶來的改變。
  • 將包裝好的Docker Image部署至AWS及Azure等雲端平台。

    • 心得感想

            這學期的實習主要定調為打基礎，老師的作法是：給我相關的電子書，並指定研讀特定的章節， 有問題再去問老師。因此從寒假開始，我看了非常多英文的書籍，書的內容也涵蓋多個領域，從一開始的Mircoservices 到時下熱門的機器學習，以及到期末的影像處理。有些領域需要邊寫程式邊理解，如：微服務的理論並不複雜，書中只用 一兩頁就介紹完其架構及目的，但是卻花費大量的篇幅在講解各種專案的微服務架構該如何搭建，從實做中去體會微服務架構的 精巧，實做中大量使用到Flask、Elasticsearch、MongoDB等相關框架、軟體，除了微服務的概念外，也順便學習了不少 額外的東西。也有些領域瞭解更多的是著重在觀念，像是：影像處理的部分，使用Scikit-Image這個套件，許多操作只需要 寥寥數行程式即可完成，但是程式碼背後的原理卻不是三言兩語可以解釋清楚的，因此在書面報告中，我也花了極大的篇幅在 記錄影像處理的部份。而在這過程中，我也遇到了意料之外的問題。
            當我知道這段期間的主要工作是閱讀大量原文書、理解各種理論時，起初 以為英文閱讀會是最大的問題，但是隨著原文書看的越來越多、專有名詞越來越熟悉後，大約在寒假後，這問題就不在那麼困擾我了， 隨即而來的問題卻是孤寂與徬徨，而這也差點擊潰我。
            因為某些因素，盧老師的實習生只剩我一人，加上創創中心正式啟用，學長姐 都移到創創中心做事，整間實驗室除了午餐時間外，大多時候只有我一人。這樣的改變，對從小習慣孤獨的我而言，是很自在的 事情，不用怕影響別人而戴耳機、可以根據自己的感覺調整冷氣溫度。起初的確有一段蜜月期，但是隨著日復一日同樣的生活， 我不經焦躁了起來。
            由於老師對於知識追根究底的態度，以及自己對歷史的偏好，影響了我 學習的方式。以影像處理為例，除了瞭解演算法的應用方式外，其背後的數學原理也花費了我大量的時間研究，我也常利用零碎的時間去搜尋 這些概念或演算法創始者的故事，老師偶爾也會在課堂上講述一些AI先驅的故事，比起原理，我對：為什麼這些人會有這樣的想法？ 更有興趣。我也會對一些開發者的情操十分折服。像是隨著對資料分析的越發瞭解，不禁佩服起Pandas作者，所有資料分析會遇到的問題， Pandas幾乎都有對應的函數可以解決。而從底下支撐起Pandas，甚至是支撐起Python的科學運算，讓Python能在資料分析領域佔有一席之地的Numpy， 也都是了不起的幕後英雄。而這幾個廣受世界上Python資料科學家愛戴的套件，背後最主要的開發者竟只有6人。比起同樣強大的Tableau等 商業軟體，這群人願意將其開源，讓所有開發者免費學習、使用，這給了我極大的震撼。
            看了這麼多的故事，從一開始的欽佩，到後面不禁多了點惆悵，到了大三才明白， 學海不僅無涯，也讓人無望。從古至今許許多多優秀的人才在各個領域發光發熱，為知識的高樓增磚添瓦，生為後輩，如今站在樓下仰望。 此時知識的高樓就好像變成了巴別塔，前人在通天的道路不斷探索、奮鬥，而後人卻因為上帝將語言打亂，而無法為巴別塔添上一塊磚。
            雖然我大學前數學都很好，但是大一不懂事，沒有學好微積分，大二不成熟，錯過了 離散數學、線性代數、統計學，這些在當時看似無用的科目，卻在大三實習時，才書到用時方恨少。看到前人為了解決問題所做的奮鬥、思辯，而我 卻連他們最後推導出的方程式都看不懂，這的確讓我有段時間非常沮喪。直到一次老師跟我們講起他的求學經歷時，才改變了我的想法。
            在沒有Google、網路不發達的年代，做研究需要每天埋首於圖書館查找書本資料， 其耗費的時間及難度可想而知，老師在那樣的時空環境下成長，到了今日互聯網發達的時代，老師也改變了學習的方法，到現在仍就不停學習。 而本業是學生的我，又怎麼還能夠淪陷在低迷的情緒中呢？
            意識到此點，我開始利用晚上的時間，去清大開放課程學習微積分，開始逐步償還 大一二欠下的知識債。的確有許多優秀的人建立了龐大的知識體系，既然前人可以在條件較差的環境下拓荒，在環境優渥的條件下，追趕前人 似乎也不再是遙不可及的事，學海的確無涯，但也讓人無限嚮往彼岸的風景，哪怕彼岸不一定存在，過程的風景也充滿者無限的魅力。

    給系上的建議

    在基礎課程或是任何課程時，可以利用開學第一週的上課時間，介紹一下該科目在各領域的應用情形及應用方式，不要只花三言兩語 帶過，在我的書面報告中，有一段在不牽涉複雜數學原理的情況下，解釋如何利用梯度尋找圖片中物體的邊緣，若是我在學習微積分 及線性代數時，能瞭解這樣的概念，當時的我或許會更有動力去念這些科目。
